 🌱 ColheCash

> Aplicativo móvel para gestão financeira de feirantes e pequenos produtores rurais.


 📌 Descrição

O **ColheCash** é uma aplicação desenvolvida com o objetivo de facilitar o controle financeiro de pequenos empreendedores, especialmente aqueles que ainda utilizam métodos informais como anotações em papel ou registros dispersos.

A solução centraliza todas as informações financeiras em um único ambiente digital, permitindo maior organização, clareza e apoio à tomada de decisão no dia a dia.

Mais do que digitalizar processos, o aplicativo foi projetado para ser **simples, acessível e intuitivo**, atendendo usuários com diferentes níveis de familiaridade com tecnologia.


 🎯 Objetivo

- Organizar o controle financeiro de pequenos negócios  
- Facilitar o registro de receitas e despesas  
- Auxiliar na análise financeira com dados visuais  
- Apoiar decisões relacionadas a custos e lucros  
- Reduzir a complexidade da gestão financeira  



 🚀 Funcionalidades

 💰 Controle Financeiro
- Registro de receitas e despesas  
- Organização por categorias  
- Histórico de movimentações  

 📊 Visualização de Dados
- Gráficos interativos (pizza e linha)  
- Acompanhamento de lucros e despesas  
- Filtros por período  

 📄 Relatórios
- Geração automática de relatórios  
- Exportação em PDF e Excel  
- Visão consolidada do desempenho financeiro  

 🧮 Precificação
- Cálculo automático de preço de venda  
- Inclusão de custos (produção, transporte, mão de obra)  
- Definição de margem de lucro  

 🔔 Alertas Inteligentes
- Notificações sobre metas e prazos  
- Avisos de variações financeiras  

 🤖 Assistente Inteligente
- Análises financeiras em linguagem simples  
- Recomendações personalizadas  



 🧑‍💻 Tecnologias Utilizadas

O desenvolvimento do **ColheCash** foi baseado em tecnologias modernas, com foco em performance, escalabilidade e experiência do usuário.



📱 Frontend — React Native (Expo)

O frontend foi desenvolvido com **React Native utilizando Expo**, permitindo criar aplicações móveis para Android e iOS com uma única base de código.

**Principais aplicações no projeto:**
- Construção das telas do aplicativo  
- Componentização da interface  
- Consumo da API REST  
- Exibição de gráficos e dados financeiros  


🔙 Backend — API REST

O backend segue o padrão **REST**, sendo responsável por toda a lógica de negócio e comunicação com o banco de dados.

**Responsabilidades:**
- Gerenciamento de usuários  
- Processamento de dados financeiros  
- Organização das informações para relatórios  
- Integração com o frontend  



 🗄️ Banco de Dados — Firestore / MySQL

O sistema utiliza banco de dados para armazenar as informações financeiras de forma segura e estruturada.

Firestore (NoSQL)
- Estrutura flexível  
- Ideal para aplicações móveis  
- Atualização em tempo real  

 MySQL (Relacional)
- Dados estruturados  
- Consultas robustas  
- Controle de relações  



🔗 Comunicação

A comunicação entre frontend e backend é feita via **requisições HTTP**, utilizando JSON.

**Principais operações:**
- GET → buscar dados  
- POST → criar registros  
- PUT → atualizar  
- DELETE → remover  


 📊 Recursos Adicionais

- Bibliotecas de gráficos interativos  
- Geração de relatórios em PDF e Excel  
- Sistema de notificações  
- Integração com inteligência artificial para recomendações  



⚙️ Metodologia

O projeto foi desenvolvido utilizando **Scrum**, permitindo evolução contínua e organização das etapas.



🧠 Boas Práticas

- Separação de responsabilidades (frontend/backend)  
- Código modular e reutilizável  
- Foco em usabilidade (UX/UI)  
- Estrutura organizada  


 🏗️ Arquitetura

O sistema segue o modelo **cliente-servidor**, garantindo:

- Escalabilidade  
- Manutenção facilitada  
- Melhor organização do código
